## Architecture

This is a Rust library (`synapse-parser`) that parses Synapse (`.syn`) files — a language-agnostic message/type definition DSL designed to generate Rust and C++ code.

### Files

- `synapse.pest` — the Pest PEG grammar (resolved via `CARGO_MANIFEST_DIR`, i.e. the project root)
- `src/lib.rs` — exposes `SynapseParser` (generated by `pest_derive`) and the `Rule` enum via `pub mod synapse`
- `examples/parse_synapse.rs` — CLI that parses a `.syn` file and prints the parse tree
- `examples/sample.syn` — full example demonstrating all language features

### Synapse language

Top-level items: `namespace`, `import`, `const`, `enum`, `struct`, `message`

```
namespace geometry

struct Point {
    x: f64 = 0.0
    y: f64 = 0.0
}

enum DriveMode { Idle = 0  Forward = 1  Error = 2 }

const MAX_SPEED: f64 = 2.5

message RobotState {
    header:      Header
    mode:        DriveMode       = DriveMode::Idle
    label:       string[<=64]    = "robot"
    sensor_data: u8[]
    waypoints:   geometry::Point[<=256]
    error_code?: i32             // optional field
}
```

**Types**: Rust-style primitives (`f32`, `f64`, `i8`–`i64`, `u8`–`u64`, `bool`, `bytes`), `string`, user-defined (`Point`, `geometry::Point`).

**Array suffixes**: `[]` dynamic, `[N]` fixed-size, `[<=N]` bounded dynamic. For `string`, `string[<=N]` means a bounded-length string (not array-of-strings).

**Fields**: `name: Type`, `name?: Type` (optional → `Option<T>`), `name: Type = value` (default).

**`struct` vs `message`**: `struct` is a plain data aggregate; `message` is a serializable unit (code generators will emit serialize/deserialize logic for messages).

### Critical PEG ordering rules

Pest uses ordered choice (`|`) with no backtracking. Two ordering constraints in `synapse.pest` must be preserved:

1. **`base_type`**: `string_type` before `primitive_type` before `type_ref`. Prevents `string` from falling through to `type_ref` and primitive names like `bool` from being matched as user identifiers.
2. **`literal`**: `float_lit` before `int_lit`. Prevents `1.0` from matching as integer `1`.
